# Frontend Standards

## Principles
- **Visual Excellence**: Interfaces must look modern, premium, and use dynamic design principles (animations, micro-interactions).
- **Component-Driven**: Build isolated, reusable components.
- **Accessibility (a11y)**: Semantic HTML and proper ARIA attributes.
- **Responsive**: Mobile-first design that scales up elegantly.

## State Management
- Use local component state for UI-only state.
- Use context/global stores for application-wide state (e.g., auth, theme).
- Keep server state (data fetching) separate from client UI state.

## Styling
- Adhere to the project's chosen design system.
- Maintain consistent design tokens (colors, typography, spacing).
- Avoid inline styles.
